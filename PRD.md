# 抖音直播冰箱咨询互动背景 MVP 产品需求文档

## 1. 文档信息

- 文档状态：MVP 基线
- 文档语言：中文
- 目标读者：产品、设计、前端、后端、测试和直播运营人员
- 上游文档：无
- 下游文档：[SPEC.md](SPEC.md)、[PLAN.md](PLAN.md)
- 文档优先级：`PRD.md` > `SPEC.md` > `PLAN.md`

当下游文档与本文件冲突时，应先修订本文件并确认产品意图，再同步修改下游文档，不允许仅在开发计划中改变需求。

## 2. 产品概述

本项目为抖音直播间提供一套在本地电脑运行的互动背景。抖音直播伴侣采集桌面中的固定直播区域，该区域包含背景图片和可交互的冰箱产品浮动 UI。

直播主持人可以坐在远离键盘的位置，通过手机、语音和手势控制直播画面。桌面同时提供不被直播采集的私有后台，用于搜索、选择产品、编辑价格和观察系统状态。

MVP 面向提供家电购买咨询的直播间，仅支持冰箱产品。

## 3. 产品目标

1. 让主持人不坐在电脑桌前也能控制冰箱产品展示。
2. 严格分离观众可见的直播区域和仅主持人可见的后台区域。
3. 支持手机、桌面、语音和手势四种互补的控制方式。
4. 在直播过程中快速选择冰箱、录入临时价格并展示参数。
5. 以本地运行、低延迟和易审计为优先目标。

## 4. 非目标

MVP 不包含：

- 主持人摄像画面的采集、编码或输出。
- 绿幕抠像和主持人与背景的合成。
- 抖音账号登录、推流或直播伴侣自动配置。
- 商品购买、下单、支付或电商库存同步。
- 在线账户、云端多租户或远程互联网访问。
- 产品目录后台编辑器或外部商品系统同步。
- 多个并行直播场次。
- 多个手机同时控制。
- 自训练手势模型。
- 对 Chrome 外部语音识别服务可用性的保证。

## 5. 用户与使用环境

### 5.1 用户角色

- 主持人：通过手机、语音或手势控制直播画面。
- 桌面操作员：可选角色，通过私有后台搜索产品、处理异常和结束场次。
- 观众：只看到直播区域中的背景、产品摘要、产品详情和动画。

### 5.2 运行环境

- 应用的前端、Python 后端、Chrome、摄像头和麦克风运行在同一台本地电脑。
- 当前可在 WSL 中编辑代码，但 WSL 与 Windows 的分隔不属于产品运行架构或验收范围。
- 桌面端以当前稳定版 Chrome 为主要浏览器。
- 手机端以当前稳定版 iPhone Safari 为主要浏览器。
- 手机和电脑连接同一个可信局域网。
- 抖音直播伴侣通过固定屏幕区域采集直播画面。

## 6. 核心用户流程

### 6.1 开播准备

1. 用户通过 Python 命令启动应用。
2. 应用显示桌面地址、手机控制地址和二维码。
3. 桌面打开工作台，确认产品目录、直播区域、摄像头和语音状态。
4. 手机扫描二维码并成为本场唯一控制手机。
5. 操作员在抖音直播伴侣中将采集区域对准工作台中的直播区域。
6. 操作员执行隐私边界检查，确认后台不在采集范围内。
7. 主持人按需启用手势控制并开始直播。

### 6.2 选择产品

1. 主持人或操作员在私有后台输入名称、型号或类型。
2. 搜索结果只在桌面后台和手机显示。
3. 选择一个产品后，直播区域立即切换为该产品。
4. 任何时候直播区域只展示一个当前产品，不显示搜索结果或完整目录。

### 6.3 输入价格

1. 主持人选择“价格”作为语音输入目标，或直接在手机/桌面输入。
2. 语音识别结果先作为私有草稿显示。
3. 松开语音按钮后进入三秒倒计时。
4. 主持人可以在倒计时内修改、确认或撤销。
5. 未操作时，三秒后自动提交可解析的合法价格。
6. 合法价格立即显示在直播区域并播放价格高亮动画。
7. 非法输入不得覆盖当前价格，并应在私有界面显示原因。

### 6.4 切换展示面板

- 摘要面板显示当前产品图片、名称、型号和直播价格。
- 详情面板显示当前产品大图和参数表。
- 任一时刻只显示其中一个面板。
- 左滑手势进入详情面板，右滑手势返回摘要面板。
- 手机和桌面按钮可执行相同操作。

### 6.5 结束场次

1. 操作员点击“结束场次”并确认。
2. 应用清除所有本场临时价格和草稿。
3. 应用关闭手势识别状态并旋转手机控制令牌。
4. 已连接手机失去控制权限。

## 7. 功能需求

### 7.1 桌面工作台

- 使用单页面分区布局。
- 包含固定比例直播区域和私有后台区域。
- 直播区域以 720×1280 为 MVP 设计坐标系，并在桌面工作台中等比缩放预览。
- 后续版本可增加输出分辨率预设选择；MVP 固定使用 720×1280。
- 直播区域边界必须清晰，外围提供安全间隔和采集辅助线。
- 私有后台必须显示：
  - 产品搜索和搜索结果。
  - 当前产品和价格编辑。
  - 当前面板和动画控制。
  - 手机连接状态。
  - 摄像头、手势和语音状态。
  - 会话结束操作。
- 后台内容不得覆盖或进入直播区域 DOM 容器。

### 7.2 直播区域

- 显示可配置背景图片。
- 未选择产品时显示适合开播等待的占位状态。
- 摘要和详情面板使用滑动过渡。
- 切换产品时保持当前面板位置。
- 未录入价格时显示“价格待定”。
- 金额以人民币格式展示。
- 支持两种动画：
  - 价格揭晓/高亮。
  - 产品聚焦。
- 动画不得永久遮挡型号、价格或关键参数。

### 7.3 产品目录与搜索

- 目录由版本控制内的本地 JSON 提供。
- 每个产品包含唯一 ID、类型、名称、型号、图片和参数表。
- 目录不保存直播临时价格。
- MVP 至少提供 6 个示例冰箱，覆盖不少于 4 种类型。
- 支持按名称、型号和类型进行不区分大小写的包含搜索。
- 搜索结果只出现在私有界面。

### 7.4 价格

- 价格属于“场次中的产品”，不属于产品主数据。
- 价格按产品 ID 分别保存。
- 接受非负人民币金额。
- 内部按整数分存储，避免浮点误差。
- 支持阿拉伯数字和常见中文金额表达。
- 一个产品的新价格只覆盖该产品的旧价格。
- 切换产品不会清除已录入价格。
- 结束场次后清除全部临时价格。
- 应用异常退出后，重启应恢复未结束场次。

### 7.5 手机控制端

- 通过桌面二维码进入。
- 使用不可预测的场次令牌授权。
- 一个场次最多允许一个活动手机连接。
- 第二个手机连接时应明确提示已被占用。
- 手机支持：
  - 搜索和选择产品。
  - 手动输入价格。
  - 选择语音输入目标。
  - 按住说话。
  - 查看、修改、确认或撤销语音草稿。
  - 切换摘要/详情面板。
  - 触发两种动画。
  - 启用或关闭手势。
- 网络短暂断开后，原手机应能在令牌仍有效时重连并恢复状态。

### 7.6 语音输入

- 语音识别运行在桌面 Chrome，使用电脑可访问的麦克风。
- 识别语言为 `zh-CN`。
- 手机长按操作远程控制桌面识别的开始和停止。
- 识别中间结果仅在私有界面显示。
- 语音目标包括“搜索”和“价格”。
- 松开后启动三秒自动提交倒计时。
- 草稿为空、识别失败或价格非法时不得自动提交。
- Chrome 不支持语音接口、权限被拒绝或服务失败时，应显示错误并保留手动输入。
- 应用不保存原始音频。

### 7.7 手势控制

- 使用独立于主持人直播画面的第二摄像头。
- 开发测试可使用笔记本内置摄像头。
- 摄像头画面和识别调试信息只出现在私有后台。
- 使用 MediaPipe 官方预训练手势识别能力判断张开手掌。
- 使用连续帧手部轨迹识别左右滑动。
- 只有在主持人显式启用后才接受手势。
- 应使用置信度门槛、最短轨迹、方向一致性和冷却时间降低误触。
- 摄像头或模型不可用时，手机和桌面按钮仍可完整控制面板。

### 7.8 会话与实时同步

- MVP 同时只支持一个本地直播场次。
- Python 后端是业务状态的唯一权威来源。
- 桌面和手机通过实时连接接收同一状态。
- 产品、价格或面板变化应在正常局域网条件下约 300ms 内反映到直播区域。
- 瞬时动画作为可重复触发的事件处理，不应被误当作持久状态。

## 8. 非功能需求

### 8.1 隐私与安全

- 手机令牌不得出现在普通日志中。
- 后端默认只面向本机和可信局域网使用。
- 所有客户端命令必须由后端校验，不信任浏览器传入状态。
- 运行时状态文件不得提交到 Git。
- 产品路径不得允许目录穿越。
- 直播区域不得渲染后台搜索词、错误堆栈、连接令牌或设备预览。

### 8.2 可靠性

- 运行时状态使用原子写入，避免进程中断产生半个 JSON 文件。
- 状态文件损坏时保留备份并安全回到新场次，不应阻止应用启动。
- 手机断线不影响桌面和直播区域。
- 摄像头、语音或手机任一能力失效，不影响桌面基本控制。

### 8.3 可审计性

- 后端和业务规则优先使用 Python 实现。
- 数据模型、金额解析、状态转换和命令校验必须有自动测试。
- 前端设备 API 通过适配器隔离，以便测试时替换。
- 不允许将关键业务规则只隐藏在 UI 组件内部。

### 8.4 离线资源

- MediaPipe 模型和 WASM 必须由本地应用提供。
- 直播运行时不依赖 CDN。
- Chrome 语音识别可能依赖浏览器供应商的网络服务，此项作为已知例外。

## 9. MVP 验收标准

- 可通过一个 Python 命令启动应用并获得桌面 URL、手机 URL 和二维码。
- 桌面工作台可加载至少 6 个示例产品并完成搜索、选择和价格录入。
- 手机可完成全部主持人控制操作，且第二个手机不能同时取得控制权。
- 直播区域只显示当前产品摘要或详情，不显示产品列表和后台信息。
- 价格在当前场次按产品保存，异常重启可恢复，结束场次后清空。
- 语音草稿支持三秒自动提交、手动确认、修改和撤销。
- MediaPipe 可识别测试夹具中的张开手掌，轨迹算法可区分左滑、右滑和无效动作。
- 所有计划步骤的自动测试均通过。
- 生产前端构建成功并可由 Python 后端提供。

## 10. MVP 后人工检查清单

以下项目不作为每一步的自动测试门禁，但在真实直播使用前必须人工验证：

- 目标电脑上的 Chrome 摄像头权限。
- 目标电脑上的 Chrome 麦克风和中文语音识别。
- iPhone Safari 局域网连接和控制体验。
- 真实摄像头下的手势阈值与误触率。
- 抖音直播伴侣固定区域采集边界。
- 目标分辨率、字体、图片和动画在直播预览中的可读性。
